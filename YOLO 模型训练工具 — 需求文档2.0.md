# YOLO 模型训练工具 — 需求文档 2.0

------

## 一、项目概述

### 1.1 项目背景

目标检测技术在游戏 UI 识别、工业质检、安防监控等领域有广泛应用。YOLO 系列模型是目前最主流的目标检测算法之一，但其训练过程依赖命令行操作和 Python 编程环境，对非技术背景用户极不友好。

当前市面上缺乏一款**开箱即用、图形化操作**的 YOLO 训练工具，用户需要在本地配置 Python 环境、安装 PyTorch 和 CUDA、编写训练脚本、调试参数……整个过程繁琐且容易出错。

**本项目旨在解决这一痛点。**

### 1.2 项目目标

打造一款 **Windows 桌面应用**，让用户通过图形界面，无需编写任何代码，即可完成 YOLO 模型的「训练 → 验证 → 导出」完整工作流。

### 1.3 一句话定位

> **"让不懂代码的人，也能轻松训练自己的 YOLO 模型。"**

### 1.4 核心设计理念

- **极致简约**：默认值即最佳实践，用户只需上传数据、上传模型、点开始
- **透明可感**：训练状态全程可见，操作后果提前告知，消除"不敢碰"的脆弱感
- **信息完整**：不因用户小白而隐藏关键数据——用户不懂时可以截图问 AI

### 1.5 版本定位

本 2.0 为**完整版本**，一次性交付全部功能，不设后续迭代计划。

## 二、用户画像

| 维度 | 描述 |
|:---|:---|

## 三、功能需求

### 3.0 页面总览

本应用共 4 个页面，通过左侧导航栏切换：

| 页面 | 图标 | 核心功能 |
|:---|:---|:---|
| **新建训练** | 📊 | 创建训练任务、配置参数、执行训练、查看日志与结果 |
| **训练历史** | 📂 | 浏览全部历史任务、查看报告、下载模型、断点续训 |
| **推理测试** | 🔍 | 加载已训练模型，对单张图片进行目标检测，查看详细结果 |
| **设置** | ⚙️ | 全局配置、系统环境检测、模型下载指引 |

**全局组件：**

| 组件 | 说明 |
|:---|:---|
| 侧边栏 | 页面导航 + 底部训练状态指示灯（🟢 空闲 / 🟡 训练中 / 🔴 异常） + 磁盘可用空间 |
| 系统环境检测 | 应用启动时自动执行，检测结果在设置页查看，异常项在侧边栏以红点提示 |
| 全局通知 | 训练完成、训练异常、环境问题的 Toast 通知（右上角弹出，3 秒自动消失）。训练进度里程碑通知（25%/50%/75%/100%）属于"信息"类型 Toast |

---

### 3.1 新建训练（核心页面）

用户完成训练任务的完整流程：

```
选择数据集 → 自动校验 → 预览标注（可选） → 选择模型文件 → 调整参数（可选） → 开始训练 → 查看结果
```

#### 3.1.1 任务基本信息

| 字段 | 类型 | 说明 |
|:---|:---|:---|
| 任务名称 | 文本输入框 | 用户自定义任务名称，默认值可为空（训练时自动生成时间戳名称） |
| 训练图集 | 文件浏览 | 支持选择本地文件夹或 ZIP 压缩包。要求包含 images/ 和 labels/ 文件夹及 data.yaml 配置文件 |
| 模型文件 | 文件浏览 | 用户自行上传 .pt 格式的 YOLO 预训练权重文件。旁边提供 `ⓘ` 图标提示"请上传 .pt 格式的 YOLO 预训练权重文件"，下方显示蓝色链接"📖 查看模型下载与版本选择指南 →"，点击跳转至设置页对应区域 |

**数据集校验（自动触发）：**
- 用户选择数据集后**自动触发**格式校验，无需等待点击按钮
- 校验中显示加载动画（转圈 + "正在校验数据集…"）
- 校验通过：绿色提示，显示"✅ 校验通过 · N 张图片 · M 个类别"
- 校验失败：红色提示，逐条列出具体问题（如"❌ 未找到 data.yaml 文件""❌ labels 目录为空""⚠️ 发现 3 个标签文件无对应图片"）
- 校验通过后才允许开始训练

**数据集预览：**
- 数据集路径旁始终显示「📷 预览数据集」按钮
- 校验通过时：预览图展示标注框叠加（含类别名）
- 校验失败或未校验时：预览仅展示图片本身，不叠加标注框（因为标签数据可能有问题）
- 点击后弹出图片查看器，展示图片样本
- 查看器功能：左右翻页浏览、缩略图网格视图（分页显示，每页 12 张，最多展示 20 张图片样本，覆盖全部类别）、显示当前图片文件名
- 预览图由后端 Python 生成（OpenCV 读取标签 TXT + 图片，绘制标注框后返回），前端以图片形式展示，不解析 YOLO 标签格式
- 目的：让用户在训练前确认数据集内容和标注质量，避免训完才发现数据有问题

**模型文件校验：**
- 用户选择 .pt 文件后自动校验文件合法性（检查文件头、最小文件大小）
- 校验失败时红色提示"该文件不是有效的 YOLO 模型文件"
- 校验通过后显示模型基本信息（如模型架构、参数数量、文件大小）

#### 3.1.2 训练参数配置

参数组织遵循"简约优先"原则：默认仅展示 4 个核心参数，其余折叠。

**核心参数（始终可见）：**

| 参数名 | 默认值 | 可选范围 | 小白话术 |
|:---|:---|:---|:---|
| 训练轮数 (Epochs) | 100 | 1 ~ 1000 | 训练多少轮：轮数越多效果可能越好，但更耗时 |
| 批次大小 (Batch Size) | 16 | 1 ~ 256 | 每批处理图片数：显存不足时可调小 |
| 图像尺寸 (Image Size) | 640 | 320 ~ 1280 | 图片输入尺寸：保持默认即可 |
| 计算设备 | auto | auto / CPU / 每张 GPU 独立列出 | 自动选择最佳设备（优先 GPU），也可手动指定 |

**设备选择的交互要求：** 下拉列表应动态显示当前电脑所有可用的计算设备：
- CPU（始终可用）
- 每张 NVIDIA 显卡分别列出，显示"显卡型号 · 显存大小 · 检测时刻可用显存（快照值）"
- 默认选中"auto（自动选择）"
- 旁设「🔄 刷新」按钮，用于热插拔显卡或驱动变更后重新检测（同时刷新可用显存快照）

**优化器参数（默认折叠，入口文字"⚙️ 优化器参数（可跳过）"）：**

| 参数名 | 默认值 | 可选范围 |
|:---|:---|:---|
| 优化器类型 | AdamW | SGD / Adam / AdamW |
| 初始学习率 | 0.001 | 0.00001 ~ 0.1 |
| 动量因子 | 0.937 | 0 ~ 1（仅 SGD 时可用，其余置灰） |
| 权重衰减 | 0.0005 | 0 ~ 0.01 |

**高级增强参数（默认折叠，入口文字"⚙️ 高级参数（可跳过）"）：**

| 参数名 | 默认值 | 说明 |
|:---|:---|:---|
| 早停耐心值 | 50 | 验证指标不再提升多少轮后自动停止训练 |
| 马赛克增强 | 开启 | 将 4 张图拼成 1 张训练，提高模型泛化能力 |
| 混合增强 | 关闭 | 两张图按比例混合训练 |
| 水平翻转概率 | 0.5 | 随机水平翻转图片的概率 |

**参数交互规范：**
- 每个参数旁统一放置 `ⓘ` 图标，hover 时显示大白话 Tooltip
- 滑块控件（适用于数值型参数）需同时显示数字输入框，两者联动
- 每个 Tooltip 需包含：该参数的作用（大白话）、默认值、建议调整场景
- 实际实现时应覆盖 Ultralytics YOLO 所支持的完整训练参数列表

#### 3.1.3 训练执行与监控

**启动训练：**
- 校验通过后「🚀 开始训练」按钮可用（否则置灰）
- 点击后执行最终检查：磁盘空间是否充足（不足则弹警告）、数据集和模型是否仍可访问

**训练中界面行为：**

| 行为 | 说明 |
|:---|:---|
| 参数区锁定 | 开始训练后，**训练参数区**显示灰色遮罩，提示"暂停训练后可修改"。需先暂停训练才能修改参数。**任务信息区（数据集和模型文件）在训练期间和暂停期间均保持锁定**——如需更换数据集或模型，需停止训练后新建任务 |
| 侧边栏状态灯 | 侧边栏底部显示 🟡 训练中，悬停显示当前任务名称 |
| 页面切换 | 用户可以正常切换到其他页面（训练历史、推理测试、设置），训练在后台继续运行 |
| 最小化/锁屏 | 训练在独立后台进程中执行，不影响前台操作 |

**实时日志：**
- 训练开始后自动展开日志终端
- 日志级别由设置页配置（详细/标准/简洁），默认"详细"
- 日志显示：时间戳 + 轮次 + loss + mAP + 学习率等关键指标
- 日志终端自动滚动到最新行，带进度条
- 进度条显示：已完成轮次 / 总轮次 + 百分比 + 预计剩余时间
- 指标表格默认展示最近 10 轮数据（表格数据区域可滚动），完整指标数据请在训练报告中查看（「📄 查看完整训练报告」→ HTML 报告包含所有轮次的完整数据）

**暂停与恢复：**
- 训练中显示「⏸ 暂停训练」按钮
- 暂停后：训练参数区解锁、日志显示"训练已暂停"、按钮变为「▶ 恢复训练」
- 暂停期间用户仅可修改**训练参数**（训练轮数、批次大小、图像尺寸、优化器参数、高级增强参数等），**不可更换数据集和模型文件**（任务信息区在暂停期间保持锁定/灰显）
- 暂停期间用户如需更换数据集或模型，需先停止训练后新建任务
- 暂停期间可关闭程序（二次确认弹窗，告知中断后果和恢复方式）

**停止训练：**
- 训练中/暂停中显示「⏹ 停止训练」按钮
- 点击后二次确认："确定要停止训练吗？已产出的模型文件（如 best.pt / last.pt）会自动保留，可通过训练历史的「断点续训」恢复。如训练尚未完成任何有效轮次、无模型文件产出，历史记录中该任务仍可点击「断点续训」从头开始。"
- 确认后优雅终止训练进程，保留已产出文件

**异常处理：**
- 训练进程意外崩溃：弹出错误通知，保留已产出的中间模型，训练历史中该任务标记为「❌ 异常中断」
- GPU 显存溢出：日志中显示具体错误信息，建议用户减小批次大小或图像尺寸

#### 3.1.4 训练结果输出

训练完成后自动展示：

| 输出项 | 格式 | 说明 |
|:---|:---|:---|
| 训练好的模型 | `.pt` 文件（best.pt / last.pt） | 可一键打开所在文件夹或直接用于推理测试 |
| 训练结果报告 | HTML | 包含训练配置摘要、损失曲线、mAP 等指标图表、验证集预测样例 |
| 指标摘要 | 页面内展示 | 最佳 mAP50、最佳 mAP50-95、总训练轮次、总耗时 |

**训练完成操作按钮：**
- 「📄 查看完整报告」— 在默认浏览器中打开 HTML 报告
- 「📁 打开模型文件夹」— 打开 best.pt 所在文件夹
- 「🔍 用此模型进行推理测试」— 跳转至推理测试页并自动加载该模型
- 「🔄 重新训练」— 保留当前配置，清空日志重新开始

### 3.2 训练历史

#### 3.2.1 功能概述

以表格形式展示所有历史训练任务，支持筛选、查看详情、断点续训、删除。

#### 3.2.2 表格字段

| 列 | 说明 |
|:---|:---|
| 任务名称 | 用户创建时填写的名称或自动生成的名称 |
| 创建时间 | 任务创建日期时间 |
| 数据集 | 数据集名称或路径摘要 |
| 状态 | 已完成 ✅ / 已停止 ⏸ / 异常中断 ❌ / 进行中 🔄 |
| mAP50 | 最佳 mAP50 指标（未完成的显示"—"） |
| 操作 | 查看报告 / 下载模型 / 断点续训 / 删除 |

> 训练历史 JSON 中每条记录除表格字段外，还需存储**完整训练配置快照**（所有参数、数据集路径、模型路径），用于断点续训时恢复全部配置。

#### 3.2.3 交互细节

**筛选与搜索：**
- 搜索框：按任务名称模糊搜索
- 状态筛选下拉框：全部 / 已完成 / 已停止 / 异常中断
- 排序：按创建时间倒序（默认）、按 mAP50 降序

**行操作：**

| 操作 | 说明 |
|:---|:---|
| 查看报告 | 在默认浏览器中打开 HTML 训练报告（仅已完成的任务可用） |
| 下载模型 | 打开 best.pt 所在文件夹 |
| 断点续训 | 仅「⏸ 已停止」和「❌ 异常中断」状态的任务可用。点击后加载上次训练的完整配置（参数、数据集路径、模型路径均从训练历史 JSON 中存储的配置快照恢复），使用 `last.pt` 作为 `--resume` 参数传入（`last.pt` 包含完整训练状态：模型权重、优化器状态、轮次计数，可从断点精确恢复），从断点继续训练 |
| 删除 | 二次确认后删除任务记录及输出文件夹全部内容（包括 best.pt / last.pt / HTML 报告等）。确认弹窗中列出待删除的文件列表，用户确认后彻底删除，不可恢复 |

**空状态：**
- 无历史记录时显示：空白插图 + "还没有训练记录" + "📊 去新建训练 →"按钮

**批量操作：**
- 支持多选行（勾选框）
- 批量删除：二次确认后调用后端批量删除接口（`delete_tasks(task_ids[])`），删除所有选中记录及关联文件

### 3.3 推理测试（新增）

#### 3.3.1 功能概述

用户加载已训练好的 YOLO 模型，上传单张图片，查看目标检测结果。设计原则：操作极简，结果详尽。

#### 3.3.2 操作流程

```
选择模型文件 → 上传/拖入图片 → 查看检测结果
```

#### 3.3.3 界面布局

自上而下分为三个区域：

**区域一：模型选择**
- 模型文件浏览按钮 + 路径显示
- 选择后显示模型基本信息（架构、参数数量、文件大小）

**区域二：图片上传**
- 拖拽上传区域（支持拖拽 + 点击上传）
- 支持格式：JPG / PNG / BMP
- 上传后显示原图缩略图

**区域三：检测结果**
- 带标注框的结果图片（框 + 类别名 + 置信度），支持缩放查看
- 检测摘要表格：每行一个检测目标，列包括"类别名、置信度、边界框坐标"
- 统计信息：总检测数、模型文件名、推理耗时、输入图片分辨率 → 推理分辨率、置信度阈值、IoU 阈值
- 操作按钮：「📥 导出结果图」「📋 导出检测数据 (JSON)」
- 导出结果图：带标注框的 PNG 格式合成图
- 导出检测数据 JSON 格式如下：

```json
{
  "model": "best.pt",
  "inference_time_ms": 34,
  "input_size": [1920, 1080],
  "inference_size": [640, 640],
  "conf_threshold": 0.25,
  "iou_threshold": 0.45,
  "detections": [
    { "class": "button", "confidence": 0.95, "bbox": [12, 34, 198, 156] },
    { "class": "icon", "confidence": 0.87, "bbox": [320, 45, 456, 189] }
  ]
}
```

#### 3.3.4 推理参数

默认折叠（入口文字"⚙️ 推理参数（可跳过）"）：

| 参数 | 默认值 | 说明 |
|:---|:---|:---|
| 置信度阈值 | 0.25 | 低于此置信度的检测结果不显示。注意：此阈值与训练时的 mAP 计算 IoU 阈值（0.5）含义不同——置信度阈值用于筛选检测结果，IoU 阈值用于判定检测框与真实框的重叠程度 |
| IoU 阈值 | 0.45 | 重叠框去重阈值，保持默认即可 |
| 图像尺寸 | 640 | 推理时缩放尺寸，保持默认即可 |

#### 3.3.5 从训练结果快捷跳转

训练完成页的「🔍 用此模型进行推理测试」按钮，点击后自动切换到推理测试页并加载该模型，用户只需拖入图片即可。

### 3.4 设置（新增）

#### 3.4.1 全局配置

| 配置项 | 类型 | 说明 |
|:---|:---|:---|
| 默认输出目录 | 文件夹浏览 | 训练结果（模型文件、HTML 报告）的默认保存位置 |
| 训练日志级别 | 下拉选择 | 详细 / 标准 / 简洁（参见 3.4.2） |

#### 3.4.2 日志级别说明

日志级别**仅影响训练时"日志终端"内文本的详细程度**。进度条和指标表格（轮次/Loss/mAP50/mAP50-95）是视觉元素，始终实时更新，不受日志级别影响。最终 HTML 报告始终包含完整数据（所有轮次的 loss 曲线、mAP 指标图表、验证集预测样例），不受日志级别影响。

| 级别 | 显示内容 | 适合场景 |
|:---|:---|:---|
| 详细（推荐） | 每轮完整的 loss、mAP、学习率等指标 + 系统信息 | 想看到完整训练过程，出问题时方便排查 |
| 标准 | 每 5 轮输出一次摘要 + 系统关键事件 | 只想大概了解训练进展 |
| 简洁 | 仅显示训练开始/结束/异常 | 挂机训练，不关心中间过程 |

#### 3.4.3 系统环境检测

- 应用启动时自动执行一次
- 设置页展示逐项检测结果（✅ 正常 / ⚠️ 警告 / ❌ 异常）
- 检测项：

| 检测项 | 说明 |
|:---|:---|
| Python 环境 | 嵌入式 Python 运行时是否就绪 |
| PyTorch | 版本号，CUDA 是否可用 |
| GPU | 每张 NVIDIA 显卡的型号、显存总量、当前可用显存 |
| 存储空间 | 系统盘和输出目录所在盘剩余空间 |

- 「🔄 重新检测」按钮用于手动刷新
- 异常项显示简要修复建议（如"未检测到 NVIDIA 驱动，将使用 CPU 训练。如需 GPU 加速，请安装 NVIDIA 驱动。"）
- 侧边栏底部用小圆点（绿/黄/红）提示整体环境状态
- **环境检测异常时**：新建训练页的「🚀 开始训练」按钮置灰，提示"请先在设置页修复环境问题"。侧边栏红点点击后跳转设置页。仅当 Python 环境、PyTorch 均正常（非磁盘空间等软警告）时，训练按钮才可用

#### 3.4.4 模型下载指引

本工具不自带模型文件。此区域提供详细指引：

- 各版本 YOLO 官方模型下载地址（YOLOv8 / YOLOv11 及后续 Ultralytics 支持的版本）
- 版本选择建议（按数据量、速度/精度偏好）
- 各后缀（n/s/m/l/x）的对比说明
- 下载后的文件存放和使用步骤

模型下载指引内容（含下载链接）在 V2.0 中硬编码于前端页面，每次版本发布时同步更新链接。

## 四、非功能需求

### 4.1 可用性

| 要求 | 说明 |
|:---|:---|
| 安装简单 | 双击安装包完成安装，无需安装任何额外软件（Python / CUDA 等全部内嵌） |
| 默认优先 | 所有参数提供合理默认值，用户可直接使用默认配置开始训练 |
| 即时反馈 | 每个操作都有明确状态提示（加载中 / 成功 / 失败），无"假死"感 |
| 一键开始 | 上传数据集 + 上传模型 + 点开始训练，三步完成 |
| 大白话沟通 | 所有参数说明、提示文字、错误信息使用通俗中文，避免专业术语 |
| 信息完整 | 训练指标、推理结果等关键数据完整展示，用户可截图向他人或 AI 求助 |

### 4.2 性能

| 要求 | 指标 |
|:---|:---|
| 应用启动时间 | < 5 秒 |
| 训练时界面响应 | 日志滚动、按钮点击、页面切换不受训练阻塞 |
| 单张图片推理 | < 2 秒（不含模型加载时间） |
| 安装包体积 | < 1.2 GB（无内置模型文件，安装后约占用 2.5GB 磁盘空间） |

### 4.3 兼容性

| 维度 | 要求 |
|:---|:---|
| 操作系统 | Windows 10 (1803+) 及 Windows 11 |
| 硬件支持 | CPU 训练（所有电脑可用）；NVIDIA GPU 加速训练（需已安装驱动） |
| Python 依赖 | 完全内嵌 Python 3.13 运行时，不依赖系统 Python 环境 |

### 4.4 安全与稳定性

| 要求 | 说明 |
|:---|:---|
| 异常隔离 | 训练进程崩溃不影响主界面，保留已产出的中间模型文件 |
| 输入校验 | 数据集（YOLO 目录结构 + data.yaml + 标签合法性）、模型文件（文件头 + 最小大小）均做合法性检查，拒绝明显无效文件 |
| 磁盘空间检查 | 训练前检查输出目录剩余空间，不足时弹出警告 |
| 训练中退出保护 | 训练进行时关闭窗口：弹出二次确认对话框，说明"训练尚未完成，退出将中断当前训练。已产出的模型文件（如 best.pt）会自动保留，之后可通过训练历史的「断点续训」恢复使用。确定要退出吗？"。若训练尚未产出任何模型文件，提示中不提及 best.pt，仅说明可从训练历史中重新开始 |
| 训练中操作锁定 | 训练开始后锁定训练参数区（灰色遮罩 + "暂停训练后可修改"）。**任务信息区（数据集和模型）在训练和暂停期间始终锁定**——更换数据集或模型需停止训练后新建任务 |
| 训练中导航保持 | 训练期间侧边栏正常可用，用户可切换到任意页面，训练在后台继续 |
| 中间产物保护 | 无论手动停止、意外崩溃、程序关闭，已产出的 best.pt / last.pt 始终保留，不会被删除。训练进程启动后即创建输出目录，后续产出的任何文件均持久化到磁盘 |

### 4.5 消除脆弱感

脆弱感定义为：用户开始训练后不敢碰电脑，生怕误操作导致训练中断或丢失。通过以下设计消除：

| 策略 | 实现方式 |
|:---|:---|
| **透明** | 侧边栏底部常驻训练状态指示灯（🟢 空闲 / 🟡 训练中 / 🔴 异常），用户切到其他页面也能感知 |
| **透明** | 训练日志持续滚动，让用户看到"程序在干活" |
| **保护** | 训练中锁定参数区，随便点也不会误改 |
| **保护** | 关闭窗口时二次确认，防止手滑 |
| **保护** | 最小化窗口、锁屏、切换页面均不影响训练（后台独立进程） |
| **恢复** | 训练历史中意外中断的任务显示「🔄 断点续训」入口 |
| **恢复** | best.pt / last.pt 始终保留，手动停止也不丢失 |

### 4.6 可维护性

- 后端训练引擎与前端 UI 解耦，便于升级 YOLO 版本
- 训练参数配置与 Ultralytics 官方接口保持同步
- 模型下载指引的链接集中管理，便于更新

## 五、技术方案

### 5.1 技术栈

| 层级 | 技术选型 | 说明 |
|:---|:---|:---|
| **桌面框架** | Tauri 2.x | 轻量级桌面应用框架，前端基于 WebView2 |
| **前端** | HTML5 + CSS3 + 原生 JavaScript | 零框架，降低复杂度和体积 |
| **后端训练引擎** | Python 3.13 + PyTorch 2.6.0+cu124 + Ultralytics 8.3.50 | 调用 YOLO 官方库执行训练和推理 |
| **Python 运行时** | 嵌入式 Python 3.13 | 打包进安装包，内含 PyTorch（CPU + CUDA 全量），不含模型文件 |
| **安装包** | NSIS | 专业 Windows 安装包，单实例运行、桌面快捷方式 |
| **任务管理** | Rust 子进程 + stdout 日志流 + metrics 文件 | 前端通过 Tauri 事件接收实时日志和训练指标 |

### 5.2 Python 后端能力矩阵

| 能力 | 调用场景 | 说明 |
|:---|:---|:---|
| 训练执行 | 新建训练 | 调用 Ultralytics train，实时输出日志流 |
| 数据集校验 | 新建训练 | 检查 YOLO 目录结构、data.yaml 完整性、标签文件合法性 |
| 模型推理 | 推理测试 | 加载模型 → 推理单张图片 → 返回检测结果 JSON（含类别、置信度、边界框、推理耗时） |
| 模型导出 | 推理测试 | 导出为 ONNX 等格式（可选） |
| 模型校验 | 新建训练 | 检查上传的 .pt 文件是否可被 Ultralytics 正常加载 |
| 环境检测 | 设置 / 全局 | 检测 Python 运行时、PyTorch 版本、CUDA 可用性、GPU 信息、磁盘空间 |
| 训练暂停/恢复 | 新建训练 / 训练历史 | 发送信号暂停/恢复训练进程，支持断点续训 |
| 报告生成 | 新建训练 | 读取训练 metrics 文件，生成 HTML 报告 |

**device 参数传递规范：** 前端下拉框值为 `"auto"` / `"cpu"` / `"cuda:0"` / `"cuda:1"` 等，Rust 层不做转换，直接透传给 Python 引擎。Python 引擎按 Ultralytics 原生格式解释：`"auto"` → 自动选择最佳设备，`"cpu"` → 使用 CPU，`"cuda:N"` → 使用指定编号的 GPU。

### 5.3 关键工程决策

**1. 日志通道与报告通道分离**

```
训练进程 stdout       → Rust 管道 → 前端日志终端（受"日志级别"控制展示量）
训练进程 metrics 文件  → Rust 读取 → HTML 报告生成（始终完整，不受日志级别影响）
```

日志是实时展示流，报告是训练完成后从完整数据生成的静态产物，两者走不同通道，互不影响。

**2. 训练与推理进程独立**

训练和推理各使用独立的 Python 子进程，避免争抢 GPU 显存。同一时间只能运行一个训练任务和一个推理任务。

**3. 数据流架构**

```
┌─────────────────────────────────────────────────┐
│  Tauri 前端 (WebView2)                          │
│  ┌───────┐ ┌────────┐ ┌────────┐ ┌────────┐   │
│  │新建训练│ │训练历史│ │推理测试│ │  设置  │   │
│  └───┬───┘ └───┬────┘ └───┬────┘ └───┬────┘   │
│      │         │          │          │         │
│  ┌───┴─────────┴──────────┴──────────┴─────┐   │
│  │        Tauri Event Bus (事件总线)        │   │
│  └──────────────────┬──────────────────────┘   │
└─────────────────────┼───────────────────────────┘
                      │ Tauri Command / Event
┌─────────────────────┼───────────────────────────┐
│  Rust 后端 (Tauri)  │                           │
│  ┌──────────────────┴──────────────────────┐   │
│  │  任务调度器                              │   │
│  │  · 训练进程管理 (spawn/stdin/stdout)    │   │
│  │  · 推理进程管理                          │   │
│  │  · 环境检测管理                          │   │
│  └──────────────────┬──────────────────────┘   │
└─────────────────────┼───────────────────────────┘
                      │ IPC (stdin/stdout/文件)
┌─────────────────────┼───────────────────────────┐
│  Python 引擎        │                           │
│  ┌──────────────────┴──────────────────────┐   │
│  │  train.py    数据集校验、训练执行、报告生成│   │
│  │  infer.py    模型推理、模型导出           │   │
│  │  env_check.py 环境检测                   │   │
│  └──────────────────────────────────────────┘   │
└─────────────────────────────────────────────────┘
```

**4. 不去内置模型的影响**

- 安装包体积预计从 ~2GB（含模型）降至 ~1.2GB
- 设置页的「模型下载指引」中的链接在 V2.0 中**硬编码于前端页面**，每次版本发布时同步更新
- 首次使用时用户需自行下载模型——这是唯一额外的步骤
