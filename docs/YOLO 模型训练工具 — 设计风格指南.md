## 一、文档说明

### 1.1 文档目的

本文档定义 YOLO 模型训练工具的整体设计风格，覆盖视觉、语言、交互三个维度。旨在确保产品界面的一致性，降低用户认知负担，让非技术背景用户获得“专业但不冷漠、可靠且易上手”的使用体验。

### 1.2 核心设计原则

本工具的设计遵循以下四条原则，所有界面决策均应回归于此：

| 原则           | 说明                                                         | 衡量标准                                                     |
| :------------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **默认即最佳** | 所有参数提供合理默认值，用户不调整也能直接开始训练           | 用户首次使用时，无需修改任何参数即可完成一次完整训练         |
| **每步有反馈** | 任何操作都有明确的状态提示（加载中/成功/失败），杜绝“假死”感 | 用户操作后 500ms 内必须有状态变化或加载指示                  |
| **危险先确认** | 不可逆操作（删除、停止、退出）必须二次确认，并告知后果       | 所有危险操作均有确认弹窗，弹窗内说明“会发生什么”             |
| **犯错能恢复** | 用户误操作后有明确的挽回路径                                 | 误停训练可通过断点续训恢复；误删任务需二次确认不可恢复（此场景无需挽回路径） |

## 二、视觉风格

### 2.1 设计定位

**“专业但友好”的暗色工具风。**

- **专业**：暗色主题、克制配色、清晰的信息层级，体现“计算密集型工具”的可靠感
- **友好**：圆润的控件、通俗的语言、及时的反馈，让小白用户不感到压迫

### 2.2 色彩体系

#### 2.2.1 调色板

| 用途             | 色值      | 说明                 | 使用场景                                 |
| :--------------- | :-------- | :------------------- | :--------------------------------------- |
| **主背景**       | `#0d1117` | 最深的背景色         | 页面主背景、侧边栏背景                   |
| **卡片背景**     | `#161b22` | 略浅于主背景         | 表单卡片、信息卡片、弹窗                 |
| **卡片内嵌背景** | `#0d1117` | 与主背景一致         | 卡片内的子区域（如日志终端、参数折叠区） |
| **分割线**       | `#30363d` | 低对比度分割线       | 页面内分割、卡片边框                     |
| **主要文字**     | `#f0f6fc` | 近白色               | 标题、正文、标签                         |
| **辅助文字**     | `#8b949e` | 中灰色               | 参数说明、时间戳、状态描述               |
| **主色调/强调**  | `#4fc3f7` | 科技蓝               | 主要按钮、链接、焦点状态、高亮信息       |
| **成功**         | `#58a6ff` | 亮蓝（替代传统绿色） | 校验通过、训练完成、可用状态             |
| **警告**         | `#d29922` | 橙黄                 | 警告信息、部分确认弹窗                   |
| **危险/错误**    | `#ff7b72` | 亮红橙               | 错误提示、删除按钮、停止按钮             |
| **特殊强调**     | `#bc8cff` | 淡紫                 | 特殊指标（如 mAP）或可选功能入口         |

#### 2.2.2 颜色使用规范

- **主按钮**：使用 `#4fc3f7` 背景，白色文字。悬停状态加深至 `#38b0e6`
- **危险操作按钮**（删除/停止）：使用 `#ff7b72` 背景或文字，配合确认弹窗
- **成功状态**：使用 `#58a6ff` 而非传统绿色，避免与“错误红”产生红绿灯联想，同时保持暗色背景下的可读性
- **文字颜色**：正文使用 `#f0f6fc`，辅助信息使用 `#8b949e`，禁用状态使用 `#484f58`

### 2.3 字体与排版

#### 2.3.1 字体家族

| 用途                      | 字体     | 备选                                           |
| :------------------------ | :------- | :--------------------------------------------- |
| **界面文字（中文）**      | 微软雅黑 | PingFang SC, -apple-system, BlinkMacSystemFont |
| **界面文字（英文/数字）** | Segoe UI | Roboto, Helvetica Neue, Arial                  |
| **代码/日志**             | Consolas | JetBrains Mono, Fira Code, monospace           |

#### 2.3.2 字号体系

| 层级        | 字号 | 行高 | 用途                       |
| :---------- | :--- | :--- | :------------------------- |
| H1 页面标题 | 24px | 1.4  | 页面主标题                 |
| H2 卡片标题 | 16px | 1.5  | 卡片/区块标题              |
| 正文        | 14px | 1.6  | 标签、说明文字、表格内容   |
| 辅助文字    | 13px | 1.5  | 提示信息、状态描述、时间戳 |
| 代码/日志   | 13px | 1.6  | 日志终端、文件路径         |
| 数值/指标   | 28px | 1.2  | 摘要卡片中的关键数字       |

#### 2.3.3 排版规范

- **标题**：字重 600（Semibold），字母间距 -0.3px
- **正文**：字重 400（Regular），字母间距 0
- **代码/日志**：字重 400，字母间距 0，背景色 `#0d1117`

### 2.4 图标系统

- **类型**：Emoji 优先（如 `📊` `⚙️` `🚀` `📂`）
- **理由**：Emoji 对小白用户最友好，无需学习图标语义即可理解
- **例外**：状态指示点（`🟢` `🟡` `🔴`）和基本操作图标（`▶` `⏸` `⏹`）
- **大小**：导航菜单使用 16px 大小（与文字同行），按钮图标使用 18px

### 2.5 间距与圆角

| 维度           | 规范        | 用途                 |
| :------------- | :---------- | :------------------- |
| **卡片内边距** | 20px ~ 24px | 表单卡片、信息卡片   |
| **组件间距**   | 16px        | 同一卡片内不同区块   |
| **页面边距**   | 40px ~ 48px | 内容区与页面边缘     |
| **圆角**       | 8px         | 按钮、输入框、小控件 |
| **圆角**       | 12px        | 卡片、弹窗、面板     |
| **圆角**       | 16px        | 主容器（最大圆角）   |

### 2.6 阴影

- **卡片**：`box-shadow: 0 2px 12px rgba(0,0,0,0.4)` — 轻微浮起感
- **弹窗**：`box-shadow: 0 8px 32px rgba(0,0,0,0.6)` — 明显浮起，强调层级
- **原则**：阴影在暗色主题下应轻微，避免过重的投影造成“脏”的感觉

## 三、语言风格

### 3.1 核心原则

**“说人话，不装专业。”**

所有界面文字应以“用户能听懂”为第一标准，而非“技术定义准确”。用户看不懂的术语，再准确也没有意义。

### 3.2 具体规则

#### 3.2.1 用“你”而不是“用户”

| ❌ 不要用                   | ✅ 要用                           |
| :------------------------- | :------------------------------- |
| 数据集校验通过，可开始训练 | 你的数据集没问题，可以开始训练了 |
| 用户需选择模型文件         | 你需要选择一个模型文件           |

#### 3.2.2 参数说明：写“大白话话术”

| ❌ 技术定义                                             | ✅ 大白话                                       |
| :----------------------------------------------------- | :--------------------------------------------- |
| 批次大小决定每次前向传播的样本数量，影响梯度估计的方差 | 每批处理多少张图：显存不够就调小一点           |
| 初始学习率控制参数更新的步长                           | 学习率：数值越大，模型学得越快，但太高可能学歪 |
| 早停耐心值：验证指标不再提升的轮数上限                 | 如果模型一直没进步，多少轮后自动停             |

#### 3.2.3 错误提示：告诉用户“怎么办”

| ❌ 只报错                         | ✅ 报错 + 建议                                               |
| :------------------------------- | :---------------------------------------------------------- |
| RuntimeError: CUDA out of memory | 显存不够用了。试试把「批次大小」从 16 改成 8                |
| FileNotFoundError: data.yaml     | 没找到 data.yaml 文件。你的数据集目录里需要包含这个配置文件 |

#### 3.2.4 按钮文字：动词 + 结果

| ❌ 模糊 | ✅ 明确                       |
| :----- | :--------------------------- |
| 确定   | 确认删除、确认停止、确认退出 |
| 提交   | 开始训练、生成报告、导出模型 |

### 3.3 语气风格参考

- **正常提示**：平和、中性。“正在校验数据集…”
- **成功反馈**：积极、肯定。“校验通过！你的数据集没问题。”
- **错误反馈**：平静、给出建议。“出了点问题：[具体原因]。试试 [解决建议]。”
- **确认弹窗**：严肃但温和。“确定要停止训练吗？已产出的模型文件会自动保留。”

## 四、交互风格

### 4.1 核心原则

| 原则             | 说明                                                     |
| :--------------- | :------------------------------------------------------- |
| **默认即最佳**   | 用户不需要调整任何参数就能跑通一次训练                   |
| **每步有反馈**   | 任何操作 500ms 内必须有状态变化或加载指示                |
| **危险先确认**   | 删除/停止/退出等操作必须二次确认，说明后果               |
| **不假死**       | 耗时操作必须显示进度（进度条 + 状态文字 + 预计剩余时间） |
| **能恢复就不怕** | 用户犯错后有明确的挽回路径                               |

### 4.2 组件交互规范

#### 4.2.1 按钮

| 类型                      | 视觉                         | 交互                                      |
| :------------------------ | :--------------------------- | :---------------------------------------- |
| **主要按钮**（Primary）   | 蓝色背景 `#4fc3f7`，白色文字 | 页面中最重要的一项操作，每个页面最多 1 个 |
| **次要按钮**（Secondary） | 透明背景，蓝色边框/文字      | 辅助操作，与主要按钮并列时弱化            |
| **危险按钮**（Danger）    | 红色文字或红色背景           | 删除/停止等操作，通常配合确认弹窗         |
| **禁用按钮**（Disabled）  | 灰色文字 `#484f58`           | 不可点击，鼠标悬停时显示禁用原因 Tooltip  |

#### 4.2.2 输入框与表单

- 输入框背景：`#0d1117`，边框 `#30363d`
- 聚焦状态：边框变为 `#4fc3f7`，无额外阴影
- 占位文字：使用 `#484f58`
- 必填项：使用红色 `*` 标记

#### 4.2.3 滑块

- 轨道颜色：`#30363d`，已填充部分：`#4fc3f7`
- 滑块圆点：`#4fc3f7`，悬停放大 1.2 倍
- 数值显示：同步显示在滑块右侧或上方

#### 4.2.4 下拉框

- 与输入框风格一致
- 选项列表背景：`#161b22`，边框 `#30363d`
- 悬停选项：背景变为 `#21262d`

#### 4.2.5 Toast 通知

- 位置：右上角固定位置，多个 Toast 垂直堆叠
- 显示时长：成功/信息 3 秒，警告/错误 5 秒
- 类型：成功（`#58a6ff`）、警告（`#d29922`）、错误（`#ff7b72`）、信息（`#4fc3f7`）

#### 4.2.6 确认弹窗

- 居中模态弹窗，背景半透明遮罩（`rgba(0,0,0,0.7)`）
- 弹窗宽度：不超过 480px
- 结构：图标（可选）+ 标题 + 详细说明 + 主/次按钮
- 说明文字需明确告知“操作后果是什么”

### 4.3 加载与进度

| 场景           | 展示方式                                                  |
| :------------- | :-------------------------------------------------------- |
| **页面加载**   | 内容区骨架屏或加载转圈                                    |
| **数据集校验** | 转圈 + “正在校验数据集…”                                  |
| **模型推理**   | 转圈 + “正在推理…”                                        |
| **训练进度**   | 进度条（百分比 + 当前/总计轮次）+ 已用时间 + 预计剩余时间 |
| **训练日志**   | 深色终端背景，新日志自动滚动到底部                        |

### 4.4 状态指示

| 状态                 | 指示方式                                           |
| :------------------- | :------------------------------------------------- |
| **环境就绪（空闲）** | 侧边栏底部 🟢 绿点 + “环境就绪”                     |
| **训练中**           | 侧边栏底部 🟡 黄点 + “训练中：[任务名]”             |
| **环境异常**         | 侧边栏底部 🔴 红点 + “N 项环境异常”，点击跳转设置页 |
| **磁盘空间不足**     | 侧边栏底部显示剩余空间，接近红线时变黄/红          |

## 五、暗色主题规范

### 5.1 设计理由

- 用户长时间盯着屏幕标注和训练，暗色主题减少视觉疲劳
- 与主流开发者工具（VS Code、GitHub、PyCharm）风格一致，提升专业感
- 暗色背景下，彩色状态指示（绿/黄/红）更加醒目

### 5.2 颜色变量（CSS 参考）

css

```
:root {
  /* 背景色 */
  --bg-primary: #0d1117;
  --bg-secondary: #161b22;
  --bg-card: #161b22;
  --bg-inset: #0d1117;
  --bg-input: #0d1117;

  /* 边框与分割 */
  --border-default: #30363d;
  --border-muted: #21262d;

  /* 文字 */
  --text-primary: #f0f6fc;
  --text-secondary: #8b949e;
  --text-muted: #484f58;
  --text-link: #4fc3f7;

  /* 状态色 */
  --color-primary: #4fc3f7;
  --color-success: #58a6ff;
  --color-warning: #d29922;
  --color-danger: #ff7b72;
  --color-purple: #bc8cff;

  /* 阴影 */
  --shadow-card: 0 2px 12px rgba(0, 0, 0, 0.4);
  --shadow-modal: 0 8px 32px rgba(0, 0, 0, 0.6);
}
```



### 5.3 深色主题适配注意事项

- 避免纯黑色（`#000000`）作为背景，使用 `#0d1117` 更柔和
- 避免纯白色（`#ffffff`）作为文字，使用 `#f0f6fc` 减少眩光
- 阴影在深色背景下应比浅色主题更柔和（降低不透明度）

## 七、一致性检查清单

在开发过程中，每完成一个页面或组件，应逐项核对：

- 背景色是否使用了 `#0d1117` 或 `#161b22`
- 主要文字是否为 `#f0f6fc`
- 辅助文字是否为 `#8b949e`
- 边框/分割线是否为 `#30363d`
- 主要按钮是否为 `#4fc3f7`
- 成功状态是否为 `#58a6ff`
- 错误/危险是否为 `#ff7b72`
- 警告是否为 `#d29922`
- 所有文字是否使用了“大白话”而非技术术语
- 所有错误提示是否包含了“怎么办”的建议
- 耗时操作是否有进度反馈
- 危险操作是否有确认弹窗
